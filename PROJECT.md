# Право olymp — полное описание проекта

> Документ для продолжения работы другой моделью/разработчиком. Здесь собрана структура, логика, история изменений и инструкции по дальнейшей разработке.

---

## 1. Цель проекта

Статический сайт (SPA) для подготовки к олимпиадам по праву, которые дают льготы при поступлении на юриспруденцию в ВШЭ и другие вузы.

Контент берётся из презентаций курса **«Общее дело»** и официальных сайтов олимпиад.

---

## 2. Технологии

- **HTML/CSS/JS**, без сборки и фреймворков.
- **GitHub Pages** для хостинга.
- **GitHub Actions** для периодической проверки сайтов олимпиад.
- **Python** для вспомогательных скриптов (генерация `data.js`, парсинг PDF, чекер олимпиад).

## 4. Что сделано

### 4.1. Базовый сайт
- SPA на чистом JS: `index.html` + `js/app.js` + `js/data.js`.
- Роутинг через `window.location.hash`.
- Мобильное меню.
- Прогресс пользователя сохраняется в `localStorage`.

### 4.2. Разделы сайта
| Раздел | Функция | Источник данных |
|---|---|---|
| **Главная** | Быстрый доступ к темам, тестам, олимпиадам, вузам | `app.js` |
| **Олимпиады** | Карточки с информацией о 15 перечневых олимпиадах | `APP_DATA.olympiads` |
| **ВУЗы** | Карточки 17 вузов, разбитые на 3 категории | `APP_DATA.universities` |
| **Темы** | 16 учебных тем по ТГП | `APP_DATA.topics` |
| **Тесты** | 16 тестов (по 1 к каждой теме) | `APP_DATA.tests` |
| **Прогресс** | Статистика изучения и результаты тестов | `localStorage` |

### 4.3. Контент
- Извлечён текст из 4 PDF-презентаций по ТГП через `pdfplumber`.
- Создано 16 тем и 16 тестов по блоку «Теория государства и права».
- Собрана информация о 15 перечневых олимпиадах с сайтов (сроки, льготы, уровни).
- Добавлены 17 вузов с категориями и примерными проходными баллами ЕГЭ.

### 4.4. Автоматизация
- **GitHub Actions workflow** `.github/workflows/check_olympiads.yml`:
  - Запускается каждый понедельник в 9:00 UTC.
  - Скрипт `scripts/check_olympiads.py` проверяет доступность сайтов олимпиад.
  - Сравнивает хеши страниц с предыдущими снепшотами.
  - Если сайт изменился или недоступен — создаёт issue в репозитории.
  - Сохраняет снепшоты в `.olympiad-snapshots.json`.

## 6. Как работает `app.js`

- **Роутинг** на `hashchange`.
- **Меню** `<a data-route="...">` обрабатывается в `initMobileMenu`.
- **Карточки** с `data-goto` обрабатываются через `bindGoto`.
- **Рендер-функции**: `renderHome`, `renderOlympiads`, `renderUniversities`, `renderTopics`, `renderTopicDetail`, `renderTests`, `renderTest`, `renderProgress`.
- **Прогресс**: сохраняется в `localStorage` (`law-prep-progress`), хранит `completedTopics` и `testResults`.

## 7. Как добавлять контент

### 7.1. Добавить тему
1. Отредактировать `js/data.js` — добавить объект в `topics`.
2. Добавить тест в `tests` с `topicId`, равным id темы.
3. Проверить валидность JSON:
   ```bash
   python3 -c "import json; data=json.loads(open('js/data.js').read()[open('js/data.js').read().index('{'):open('js/data.js').read().rfind('};')+1]); print('OK')"
   ```

### 7.2. Добавить олимпиаду
1. Добавить объект в `APP_DATA.olympiads` в `js/data.js`.
2. Обязательные поля: `id`, `name`, `subject`, `organizer`, `level`, `status`, `grades`, `stages`, `benefits`, `format`, `links`.
3. Для универсальных формулировок используй: «Сезон завершён. Регистрация на следующий сезон обычно открывается...».

### 7.3. Добавить вуз
1. Добавить объект в `APP_DATA.universities`.
2. Обязательные поля: `id`, `name`, `city`, `program`, `exams`, `olympiads`, `site`, `note`, `tier`, `budgetEge`.

### 7.4. Генераторы (если нужно)
- `data/part1.py`, `data/part2.py`, `data/part3.py` — части данных.
- `generate_data_full.py` — собирает `js/data.js` из частей.
- Запуск:
  ```bash
  python3 generate_data_full.py
  ```

## 8. GitHub Actions

Файл: `.github/workflows/check_olympiads.yml`

**Что делает:**
1. Запускается по понедельникам в 9:00 UTC или вручную.
2. Устанавливает Python и зависимости (`requests`).
3. Запускает `scripts/check_olympiads.py`.
4. Коммитит обновлённый `.olympiad-snapshots.json`.

**Как включить:**
- В репозитории GitHub: **Settings → Actions → General** → разрешить Actions.
- Во вкладке **Actions** включить workflow «Проверка олимпиад».

**Логика скрипта:**
- Загружает главные страницы сайтов олимпиад.
- Сохраняет хеши нормализованного HTML.
- Сравнивает с предыдущими снепшотами.
- Если хеш изменился или сайт недоступен — создаёт issue с меткой `olympiad-check`.

## 10. Важные замечания

- **PDF/слайды не в git** — `.gitignore` исключает `assets/pdfs/` и `assets/slides/`.
- **Данные в `js/data.js`** — единственный источник контента для сайта.
- **Проходные баллы ЕГЭ** — примерные, для ориентира. Точные цифры нужно смотреть на сайтах вузов.
- **Статусы олимпиад** — не привязаны к конкретной дате, чтобы сайт оставался актуальным долго.
- **Не официальный ресурс** — платформа создаётся для личного использования, не связана с ВШЭ.
